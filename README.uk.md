# ProPresenter Lyrics Formatter

[English](README.md) | [Русский](README.ru.md) | **Українська**

Швидкий інструмент, який перетворює вставлений текст пісні у формат слайдів ProPresenter — секції визначаються автоматично, форматування опційне, а результат готовий одразу для вставки в ProPresenter.

## Що вміє

- Вставляйте текст пісні як є — з будь-якого сайту, чату, документа — без попереднього форматування.
- Автоматично розпізнає назви секцій — **Verse (Куплет), Chorus (Приспів), Bridge (Бридж), PreChorus (Перед приспів), Tag, Intro (Інтро), Ending (Концовка), Outro (Аутро), Interlude (Програш), Vamp (Вамп), Turnaround (Перехід), Blank (Порожньо)** — англійською, російською та українською, включно з поширеними одруками та варіантами зі змішаною розкладкою.
- Заголовки секцій не обов'язково мають бути відокремлені порожніми рядками: застосунок так само коректно працює, якщо весь текст вставлений одним суцільним блоком без пропусків між секціями.
- Розбиває текст на слайди за кожним перенесенням рядка — один рядок тексту пісні стає одним слайдом ProPresenter.
- Опційне форматування кожного слайда: велика перша літера, видалення розділових знаків, видалення посилань.
- Початковий непозначений блок (зазвичай назва пісні/автор) автоматично отримує префікс `Title: ` у скопійованому результаті — якщо в пісні є хоча б одна розпізнана секція.
- Живий перегляд показує кожен слайд окремою карткою з рамкою, забарвленою в колір секції (Verse — сині відтінки, Chorus — малиновий/рожево-червоний, Bridge — фіолетовий тощо), і назвою секції на кожній картці — так одразу видно, чи правильно пісня розбилася на секції.
- Усе працює прямо в браузері. Жодного бекенду, акаунтів і відправки даних кудись.

## Швидкий старт

```bash
npm install
npm run dev         # запустити dev-сервер
npm run build        # перевірка типів і збірка для продакшену
npm run test          # запустити тести
npm run lint          # eslint
npm run typecheck     # tsc --noEmit
npm run format         # prettier --write
```

## Додавання UI-компонентів

Проєкт використовує shadcn/ui — нові компоненти додаються через CLI, а не пишуться вручну:

```bash
npx shadcn@latest add button
```

Імпортуються як зазвичай:

```tsx
import { Button } from "@/components/ui/button"
```
