# ProPresenter Lyrics Formatter

[English](README.md) | **Русский** | [Українська](README.uk.md)

Быстрый инструмент, который приводит вставленный текст песни к формату слайдов ProPresenter — секции определяются автоматически, форматирование опционально, а результат готов к вставке прямо в ProPresenter.

## Что умеет

- Вставляйте текст песни как есть — с любого сайта, из чата, документа — без предварительного форматирования.
- Автоматически распознаёт названия секций — **Verse (Куплет), Chorus (Припев), Bridge (Бридж), PreChorus (Пред-припев), Tag, Intro (Интро), Ending (Концовка), Outro (Аутро), Interlude (Проигрыш), Vamp (Вамп), Turnaround (Переход), Blank (Пусто)** — на английском, русском и украинском, включая распространённые опечатки и варианты со смешанной раскладкой.
- Заголовки секций не обязаны быть отделены пустыми строками: приложение так же корректно работает, если весь текст вставлен одним сплошным блоком без пропусков между секциями.
- Разбивает текст на слайды по каждому переносу строки — одна строка текста песни становится одним слайдом ProPresenter.
- Опциональное форматирование каждого слайда: заглавная первая буква, удаление знаков препинания, удаление ссылок.
- Начальный неразмеченный блок (обычно название песни/автор) автоматически получает префикс `Title: ` в скопированном результате — если в песне есть хотя бы одна распознанная секция.
- Живой предпросмотр отображает каждый слайд отдельной карточкой с рамкой, окрашенной в цвет секции (Verse — синие тона, Chorus — малиновый/розово-красный, Bridge — фиолетовый и т.д.), и названием секции на каждой карточке — так сразу видно, правильно ли песня разбилась на секции.
- Всё работает прямо в браузере. Никакого бэкенда, аккаунтов и отправки данных куда-либо.

## Быстрый старт

```bash
npm install
npm run dev         # запустить dev-сервер
npm run build        # проверка типов и сборка для продакшена
npm run test          # запустить тесты
npm run lint          # eslint
npm run typecheck     # tsc --noEmit
npm run format         # prettier --write
```

## Добавление UI-компонентов

Проект использует shadcn/ui — новые компоненты добавляются через CLI, а не пишутся вручную:

```bash
npx shadcn@latest add button
```

Импортируются как обычно:

```tsx
import { Button } from "@/components/ui/button"
```
